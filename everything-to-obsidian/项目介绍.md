# 收藏夹内容不吃灰 —— 用 AI 开发浏览器插件，打造个人知识库闭环

> 一个关于「信息焦虑 → 知识沉淀 → 内容产出」的实践故事

---

## 一、问题的起点：收藏夹是信息的坟墓

相信很多人都有这样的经历：

- 看到一篇好文章，收藏了，然后再也没打开过
- 浏览器收藏夹里有几百条书签，却不知道里面有什么
- 想找某个知识点，明明"收藏过"，就是找不到
- 知道要整理，但整理本身需要花大量时间，一直拖着

**收藏夹本质上是「信息焦虑的垃圾桶」**——我们用收藏来缓解"错过好内容"的焦虑，却从不真正消化这些内容。

信息不流动，就不会变成知识；知识不沉淀，就无法变成生产力。

---

## 二、解法：构建「抓取 → 沉淀 → 检索 → 产出」闭环

在 AI 时代，我们已经有了非常成熟的个人知识库工具——**Obsidian**。

Obsidian 的核心优势：
- 本地存储，数据完全自己掌控
- Markdown 格式，兼容性极强
- 双链笔记，构建知识网络
- 与 AI 工具（如 Claude、GPT）天然集成，可以基于知识库进行问答和内容生成

但问题在于：**把网页内容搬进 Obsidian，是一件非常费力的事情。**

手动复制、粘贴、整理格式……这个摩擦成本，让大多数人放弃了坚持。

**解法就是：用 AI 开发一个浏览器插件，把「抓取」这一步变成一键操作。**

---

## 三、项目介绍：Everything to Obsidian

> 🐝 **Everything to Obsidian** —— 将网页、收藏夹批量提取为 Markdown，一键导入本地 Obsidian 知识库

### 项目背景

本插件由 AI（Claude）辅助开发，从需求分析、技术选型、代码实现到 UI 设计，全程采用 **OpenSpec 规范驱动开发**工作流，完整记录了设计决策和实现过程。

整个开发过程验证了一个核心观点：**在 AI 时代，有产品思维的人可以快速将想法变为可用工具，不再受限于编码能力。**

### 技术栈

| 技术 | 用途 |
|------|------|
| Chrome Extension Manifest V3 | 插件框架，Service Worker 后台保活 |
| File System Access API | 直接读写本地文件系统，无需服务器 |
| Readability.js | 智能提取网页正文，过滤广告和导航 |
| Turndown.js | HTML → Markdown 高质量转换 |
| IndexedDB | 持久化目录授权句柄，无需重复授权 |
| chrome.storage.session | 跨窗口实时状态同步（取消信号） |

## 四、核心功能

### 功能一：一键保存当前网页

打开任意网页，点击插件图标，选择提取深度，一键保存为 Markdown 文件到 Obsidian 目录。

**生成的 Markdown 包含：**

```markdown
---
title: "文章标题"
source: "https://原始链接"
author: "作者"
description: "页面描述"
tags: []
dateSaved: "2026-04-09"
---

# 文章标题

（完整正文 / 摘要，由 Readability.js 智能提取）
```

**亮点：**
- Readability.js 自动过滤广告、导航、评论区，只保留核心内容
- 文件名自动 sanitize，非法字符自动处理
- 同名文件直接覆盖，不产生重复

---

### 功能二：批量导入收藏夹

选择收藏夹中的文件夹（支持多选），一键批量抓取所有书签页面内容，保存为 Markdown。

**亮点：**
- 树形展示收藏夹结构，整行可点击，勾选父目录自动递归选中子目录和书签
- 支持单独勾选某个书签，不依赖文件夹批量勾选
- **保持收藏夹目录结构**：`收藏夹/技术/前端/React入门.md`，不平铺
- 3 个并发处理，带超时跳过机制（死链、需登录的页面自动跳过）
- **独立进度窗口**：导入过程中可自由切换标签页，不会因点击空白关闭

**进度窗口实时展示：**
```
📥 批量导入中…

████████████████░░  72 / 86
⏳ 正在处理：某篇文章标题...

✅ React 入门指南
✅ Vue3 官方文档
⏭️ 某 Google 表格（超时 >15s）
❌ 某死链（页面不存在）
```

---

### 功能三：灵活的目录配置

在设置页面（Options）中：

- **授权读写目录**：一次授权，永久有效（基于 File System Access API + IndexedDB）
- **子目录下拉选择**：自动读取已授权目录下的所有子文件夹，下拉选择，无需手动输入
- **新建目录**：直接在插件内创建新子目录，实时刷新下拉列表

## 五、使用流程

```
第一步：安装插件
  → Chrome 扩展管理页加载本地文件夹

第二步：授权目录（一次性）
  → 打开设置页 → 点击「选择授权目录」→ 选择 Obsidian vault 文件夹
  → 选择保存到哪个子目录（如 Clippings）

第三步：保存当前网页
  → 打开任意网页 → 点击工具栏插件图标
  → 选择「摘要」或「完整正文」→ 点击「立即保存」
  → ✅ 文件已写入 Obsidian 目录

第四步：批量导入收藏夹
  → 点击插件图标 → 在「批量导入收藏夹」区域选择文件夹
  → 点击「开始导入」→ 弹出独立进度窗口
  → 等待完成，查看导入汇总

第五步：在 Obsidian 中使用
  → 用 Obsidian 打开 vault → 看到新增的 Markdown 文件
  → 用双链建立知识关联
  → 结合 AI 插件（Smart Connections 等）进行知识问答和内容生成
```

## 六、AI 时代的意义：知识库 × AI = 内容产出飞轮

构建完个人知识库之后，真正的价值才开始释放：

```
收藏夹/网页
    ↓ Everything to Obsidian（一键抓取）
Obsidian 知识库
    ↓ AI 插件 / 导出给 LLM
个性化知识问答 & 内容生成
    ↓
工作产出（报告 / 方案 / 文章）
    ↑
持续沉淀，飞轮转动
```

**具体场景举例：**

- 整理了 50 篇关于「用户增长」的文章 → 让 AI 基于这些文章帮你写增长方案
- 积累了行业竞品分析 → 让 AI 对比总结，生成竞争力报告
- 保存了大量技术文档 → 随时向 AI 提问，精准定位答案
- 长期积累个人笔记 → 让 AI 发现知识盲区和关联

**信息的价值在于流动和连接，而不是收藏本身。**

## 七、开发感悟

这个插件从想法到可用版本，**核心开发周期不到 2 天**，全程由 AI 辅助完成。

这个过程让我深刻体会到：

1. **AI 是放大器，不是替代者**：有清晰的需求和产品思维，AI 可以帮你快速落地；没有方向，AI 也只会产出噪音
2. **规范驱动开发的价值**：用 OpenSpec 工作流记录每一个设计决策，让 AI 始终在正确的轨道上工作，也让项目有了可追溯的"设计文档"
3. **工具的意义是降低摩擦**：一个好工具，不是增加你的工作量，而是让你愿意坚持做某件有价值的事
